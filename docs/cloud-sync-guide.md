# Life OS 云端同步功能指南

## 🎯 功能概述

Life OS 现在支持**云端同步**功能，你的数据会自动在多个设备间同步：
- 习惯打卡数据
- 日记记录
- 目标进度
- 财务记录
- 图书馆信息
- 旅行计划
- 所有其他应用数据

## 🔄 同步原理

1. **本地优先**：数据先保存在浏览器 localStorage
2. **后台同步**：自动异步同步到 Supabase 云端数据库
3. **多设备同步**：所有设备访问同一个云端数据库
4. **离线可用**：网络断开时使用本地数据，恢复后自动同步

### 步骤1：访问设置页面
1. 打开 https://carsonyang3-byte.github.io/doris-life-os/
2. 点击顶部的 **Settings** 标签页

### 步骤2：设置 Supabase 数据库
1. 你需要创建一个 Supabase 账户（免费）
2. 创建新项目
3. 执行以下 SQL 创建表：
   ```sql
   -- 文件在项目根目录: supabase_setup.sql
   -- 复制到 Supabase SQL Editor 中执行
   ```

4. 获取 API 密钥：
   - 进入项目设置 → API
   - 复制 `URL` 和 `anon public key`

5. 更新本地 .env 文件：
   ```
   VITE_SUPABASE_URL=你的项目URL
   VITE_SUPABASE_KEY=你的anon public key
   ```

### 步骤3：测试同步
1. 在设置页面点击「立即同步」
2. 查看同步状态
3. 在不同设备上访问，数据会自动同步

## ⚙️ 设置页面功能

### 云端同步卡片
- **立即同步**：手动触发数据同步
- **导出备份**：下载所有数据的 JSON 备份文件
- **导入备份**：从备份文件恢复数据

### 存储信息卡片
- 查看本地数据量
- 检查云端连接状态
- 查看最后同步时间

### 账户操作卡片
- **清除本地缓存**：删除本地数据，从云端重新加载
- **退出登录**：清除登录状态，重新输入密码

## 🔧 手动同步流程

### 从新设备同步到旧设备
1. **在新设备上**（有最新数据的）：
   - 进入 Settings → 点击「导出备份」
   - 保存 JSON 文件

2. **在旧设备上**：
   - 进入 Settings → 在「导入备份」区域
   - 粘贴 JSON 内容或上传文件
   - 点击「导入备份」
   - 点击「立即同步」

### 解决数据冲突
如果多个设备同时修改了同一数据：
- 最后修改的时间戳优先
- 系统使用 `updated_at` 字段决定哪个版本生效

## 📱 多设备使用

### 支持设备
- PC（Chrome, Edge, Safari, Firefox）
- 手机（iOS Safari, Android Chrome）
- 平板电脑

### 登录状态
- 登录状态保存在 sessionStorage
- 关闭浏览器需要重新输入密码
- 密码不保存在云端（只存在本地）

## 🛠️ 故障排除

### 常见问题

#### Q1: 同步失败
- 检查网络连接
- 确认 Supabase 项目没有过期
- 检查 .env 文件中的 API 密钥

#### Q2: 数据不一致
- 点击「立即同步」强制刷新
- 如果问题持续，清除本地缓存后重新同步

#### Q3: 云端连接错误
- Supabase 项目可能已暂停（免费计划有使用限制）
- 检查控制台错误信息
- 使用「导出备份」功能保存数据

#### Q4: 手机上看不到更新
- 确保手机浏览器访问的是最新版本
- 强制刷新：长按刷新按钮 → "强制刷新"
- 清除浏览器缓存

### 调试方法
1. 按 F12 打开开发者工具
2. 查看 Console 标签页的日志
3. 检查是否有网络错误
4. 查看 Application → LocalStorage 中的数据

## 💾 数据备份策略

### 自动备份
- 每次修改自动同步到云端
- Supabase 提供每日备份（免费计划）

### 手动备份
- 建议每周导出一次备份文件
- 保存在安全的地方（如 Google Drive）

### 数据恢复
1. 从备份文件导入
2. 系统会自动合并数据
3. 冲突数据以备份文件为准

## 🔒 安全说明

### 数据安全
- 所有数据在传输时加密（HTTPS）
- 数据库使用行级安全（RLS）
- 密码只保存在本地，不传输到云端

### 隐私保护
- 所有数据属于单个用户
- 无用户间数据共享
- 可随时导出并删除云端数据

## 📈 性能优化

### 网络使用
- 增量同步：只传输变化的数据
- 后台同步：不阻塞用户界面
- 失败重试：网络错误时自动重试

### 存储效率
- 数据压缩存储
- 定期清理过期数据（可选）
- 智能缓存策略

## 🔄 迁移指南

### 从纯本地版迁移
1. 在现有设备上导出备份
2. 设置 Supabase 数据库
3. 在新设备上导入备份
4. 数据自动同步到云端

### 切换设备
1. 在旧设备上导出备份
2. 在新设备上设置 Life OS
3. 导入备份
4. 所有数据自动可用
